# 个性化学习契约所需字段清单（待主线冻结）

状态：仅供主线契约设计使用，不是已发布 API。iOS 当前没有为以下字段创建 DTO、端点或请求，也不会探测未公布服务。

## 1. 能力协商

- 功能是否可用、不可用原因、只读/可写状态。
- 契约名称与版本、端点由服务端能力协商返回，不由客户端猜测。
- 支持的反馈动作、幂等请求头、并发控制字段及分页方式。
- 模型是否可提出建议、是否允许自动接受；必须明确 `AI 不写原始经营事项`。

## 2. 隔离与权限范围

- 组织、账套、当前用户及角色的稳定标识。
- 可读/可反馈/可管理学习结果的独立权限。
- 数据地域、保留期限、撤回或清除个性化记忆的能力。
- 服务端审核人身份；客户端不得自行填充可信审核人。

## 3. 经营事项引用

- `recordId`、经营事项版本、分类版本、事实哈希。
- 业务日期、收支方向、币种和 decimal-string 金额。
- 当前闭集分类代码与分类表版本、规范化归并名、稳定分组键。
- 仅引用原始事实，不复制或合并不可追溯的原始记录。

## 4. 建议与证据

- 建议标识、产生方式（确定性规则/模型/人工历史）、建议版本。
- 候选闭集分类、规范化归并名、同类分组键。
- 置信度、原因代码、证据引用、适用范围和生成时间。
- 使用的规则/模型/提示配置版本，便于审计和回滚。
- `modelCanAccept=false`、`writesBusinessRecords=false` 等安全声明。

## 5. 人工反馈事件

- 反馈事件标识、幂等键、动作（接受/更正/拒绝）。
- 原建议值与人工最终值、结构化原因代码及必要的补充说明。
- 提交时的经营事项版本、分类版本和事实哈希。
- 服务端审核人、角色、提交时间、处理结果和冲突详情。
- Harness 状态必须继续保留 accepted/review/rejected，rejected 必须失败关闭。

## 6. 学习结果与解释

- 个性化配置/规则集标识及版本、作用范围（用户/账套/组织）。
- 学到的偏好应引用反馈事件，不直接改写历史经营事实。
- 本次建议命中的历史反馈/规则引用、未采用原因和可撤回入口。
- 生效时间、失效时间、最后更新人及审计事件标识。
- 删除源记录或撤回反馈后，学习结果的失效与重算状态。

## 7. 同步和冲突

- 稳定幂等键、请求体指纹、重试状态。
- expected record/classification/profile version 与可识别的 409 冲突。
- 游标必须不透明；客户端只保存翻页栈，不解析游标内容。
- 离线草稿、未提交理由和本地保存意图不能被刷新覆盖。

## 主线冻结前的 iOS 边界

- 不增加个性化学习 API Client、DTO 或界面写入口。
- 不根据现有分类字段推断未来个性化接口。
- `AppSession` 继续只通过带 If-Match 的 `/api/state` 同步。
- 分类建议、同类归并和模型结果不能直接写 `BusinessRecord`。
- 只有能力协商明确公布且安全字段匹配后，客户端才允许请求对应功能。
